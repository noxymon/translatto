Overlay Alignment & Block Stacking Options

Select the preferred design approaches to resolve the translation shift and text stacking issues.

1. Overlay Alignment & Status Bar Cropping

To fix the layout shift where translation boxes do not align with original Japanese text, we propose matching coordinate spaces by cropping the status bar from screenshots and letting the overlay start directly below it.

A

Option A (Recommended): Crop Status Bar + Standard Overlay

In Kotlin, retrieve the system status_bar_height and crop it from the screenshot before OCR. The overlay window also starts below the status bar, resulting in a perfect 1-to-1 coordinate match without any complex calculations or status bar window flags.

  • Perfect, pixel-exact text alignment on all devices
  • Status bar elements (time/wifi) are excluded from OCR
  • Very simple mathematical logic (uniform scaling)
  • If a device has a large navigation bar, the very bottom might have a tiny offset
B

Option B: Fullscreen Overlay (Covers Status Bar) + No Cropping

Do not crop the screenshot, but set window layout flags in Android to force the overlay window to extend into the status bar area (fullscreen).

  • Covers the entire screen area
  • Hard to draw over status bar reliably on all Android versions
  • Notches/cutouts make drawing text at the very top problematic

2. Preventing Stacked / Overlapping Translations

When English translations are longer than the original Japanese blocks, they expand and overlap vertically. We propose two approaches to prevent this.

A

Option A (Recommended): Pre-Merge Adjacent OCR Blocks

Before translating, merge OCR blocks that belong to the same paragraph or line (based on close vertical distance and horizontal alignment). Then translate and render the merged block as a single box.

  • Improves translation quality dramatically (Gemma receives full sentences)
  • Naturally prevents overlapping/stacking of adjacent text lines
  • Fewer boxes render, making the screen look clean and premium
  • Requires a custom grouping algorithm in Dart
B

Option B: Shift Non-Merged Blocks at Render-Time

Keep blocks separate, but adjust coordinates dynamically during painting to push overlapping boxes downwards.

  • Keeps translation calls simple (no grouping logic)
  • Translating fragmented lines yields lower quality translations
  • Shifting boxes away from original text makes them hard to associate

3. Hiding the FAB during Screenshots

To prevent the floating overlay trigger button from appearing in the screen capture, we will resize the overlay window to 1x1 pixel immediately upon clicking, wait 100ms for the Android window manager to refresh, take the screenshot, and then restore it to full screen (on success) or 140x140 (on error/no-text).
