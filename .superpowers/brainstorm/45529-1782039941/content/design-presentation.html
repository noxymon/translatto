Component 1: Kotlin Status Bar Cropping (Android)

Modify processImageAndReply inside MediaProjectionService.kt to calculate the physical status bar height and crop it from the screenshot before saving.

Kotlin Execution Logic
// 1. Resolve status bar height in physical pixels
val resourceId = resources.getIdentifier("status_bar_height", "dimen", "android")
val statusBarHeight = if (resourceId > 0) resources.getDimensionPixelSize(resourceId) else 0

// 2. Crop bitmap vertically
val cleanBitmap = if (rowPadding == 0) {
    if (statusBarHeight > 0 && statusBarHeight < height) {
        Bitmap.createBitmap(bitmap, 0, statusBarHeight, width, height - statusBarHeight)
    } else bitmap
} else {
    val cropY = if (statusBarHeight > 0 && statusBarHeight < height) statusBarHeight else 0
    val cropHeight = if (statusBarHeight > 0 && statusBarHeight < height) height - statusBarHeight else height
    Bitmap.createBitmap(bitmap, 0, cropY, width, cropHeight)
}

Component 2: Dart OCR Block Merging (Flutter Main Isolate)

Before translating, we group and merge OCR blocks that represent lines of the same paragraph or words on the same line to avoid overlapping boxes and improve translation coherence.

Block Merging Criteria
  • Geometrical Overlap: If two bounding boxes overlap directly (rectA.overlaps(rectB)), they are merged.
  • Vertical Stacking (Paragraphs): If blocks are aligned horizontally (horizontal overlap is > 30% of the smaller block's width) and the vertical gap between them is small (< 1.5x the smaller block's height), they are merged.
  • Horizontal Alignment (Words): If blocks are on the same line (vertical overlap is > 50% of the smaller block's height) and the horizontal gap is small (< 2.0x the smaller block's height), they are merged.
  • Text Concatenation: Merged blocks are concatenated with spaces (e.g. "${b1.text} ${b2.text}") and bounded by a single combined rectangle (b1.boundingBox.expandToInclude(b2.boundingBox)).

Component 3: FAB Visibility and Isolate Coordination

Coordinate the overlay resize state and the capture triggers in lib/main.dart.

State Transitions Flow
  1. Trigger: User taps the FAB in OverlayWindowScreen.
  2. Hide FAB: Resize overlay window to 1x1 pixels via FlutterOverlayWindow.resizeOverlay(1, 1, false) and delay 100ms.
  3. Capture: Send "capture" request to the main isolate via the custom bridge.
  4. Restore / Render:
    • On success: Resize overlay to full screen (imageWidth / devicePixelRatio by imageHeight / devicePixelRatio) and paint translations.
    • On error or no_text: Resize overlay back to 140x140 trigger window and show error feedback if applicable.
