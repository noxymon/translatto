Design Section 2: Component Breakdown & Packages

Core Flutter packages and platform APIs utilized

Component Stack

1. Screen Capture (Screenshare)

Uses **MediaProjection API** (standard screenshare permission dialog on Android). Managed via native Kotlin platform channel or package.

Package: Custom platform channel invoking `MediaProjectionManager` and `ImageReader` to capture clean screen bitmaps.

2. Text Recognition (OCR)

Processes the captured bitmap to extract text and precise screen coordinates (left, top, width, height) of Japanese characters.

Package: `google_mlkit_text_recognition: ^0.13.0` (supports local high-accuracy Japanese OCR model).

3. Local Translation (LLM)

Runs the quantized Gemma model offline on the mobile CPU/GPU. Passes Japanese sentences, returns translation.

Package: `flutter_gemma: ^1.0.1` (wraps MediaPipe LLM Inference). Model downloaded to local App Documents directory.

4. System Window Overlay

Renders the transparent AR overlay on top of active games, manga readers, or other apps. Draws translations exactly in position.

Package: `flutter_overlay_window: ^0.4.0` (or custom Kotlin system alert overlay window for performance).

Key Dependency Flow

flutter_gemma (Inference Engine) ➔ Needs gemma-2b-it-cpu-int4.bin model file in App Docs folder.
google_mlkit_text_recognition ➔ Automatically downloads Japanese OCR model on first run.
