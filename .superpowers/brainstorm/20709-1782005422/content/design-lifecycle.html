Design Section 3: Lifecycle & Error Handling

Managing memory, performance, and runtime errors in Flutter

Lifecycle & Resource Management

1. Memory Management (Gemma 4 Lifecycle)

Gemma model files are large (~1.5GB+). To prevent OS background process kills (OOM):
• Lazy Load: Model only initialized when user activates overlay.
• Dispose: Release model resources when overlay is dismissed by user.

2. Permissions Flow

• Overlay: Request system overlay permission (leads user to Settings page).
• Screenshare: MediaProjection permission requested each session start or persistently if allowed by OS.

3. Error States

• Missing Model: If Gemma `.bin` file is missing in App Docs, show download setup guide in main app UI.
• Zero OCR Hits: If ML Kit finds no Japanese characters, show a brief "No text found" overlay toast and auto-dismiss.
• High Latency Spinner: Display a subtle loading animation in the overlay boxes while `flutter_gemma` processes translation.
