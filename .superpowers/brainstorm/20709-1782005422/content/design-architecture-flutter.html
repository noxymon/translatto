Design Section 1: Flutter + flutter_gemma Architecture

Hybrid Flutter-Kotlin architecture for Android Screen Translator

Data & Component Flow

1. Native Kotlin Service

Runs background service for **Floating Action Button Overlay** and **MediaProjection API**. Captures screen bitmap on click, passes it to Flutter via **Platform Channels**.

2. Google ML Kit OCR (Dart)

Flutter receives bitmap. Runs offline `google_mlkit_text_recognition` to detect Japanese characters and get their bounding box coordinates.

3. Local Gemma 4 via flutter_gemma

Feeds extracted Japanese strings into `flutter_gemma` package. It uses MediaPipe LLM Inference SDK to run Gemma locally on-device and return English text.

4. AR Overlay Renderer

Draws translations directly on the overlay window at exact Japanese coordinates using Flutter CustomPainter (overlay window hosted by package like `flutter_overlay_window`).
